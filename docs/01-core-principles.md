# 01｜系统综述与Meta分析核心流程

## 1. 总流程

一篇合格的系统综述/Meta分析至少包含以下闭环：

1. 提出结构化问题：PICO/PECO/PIRD/PIRT。
2. 制定并注册protocol。
3. 设计可复现检索策略。
4. 双人独立筛选文献。
5. 双人独立提取数据。
6. 评估偏倚风险。
7. 判断是否适合定量合成。
8. 进行主分析、异质性分析、敏感性分析、亚组/Meta回归。
9. 评估小样本效应/发表偏倚/选择性报告。
10. 分级证据确定性。
11. 按PRISMA或相应扩展报告。
12. 开放数据、代码、检索式和分析日志。

## 2. 问题构建

### 2.1 干预问题：PICO

- Population：疾病、年龄、性别、分期、基线风险、临床场景。
- Intervention：药物、剂量、疗程、联合治疗、手术方式、护理措施。
- Comparator：安慰剂、常规治疗、其他干预、无干预。
- Outcome：患者重要结局，而不只是替代指标。

### 2.2 暴露/危险因素问题：PECO

- Population：目标人群。
- Exposure：暴露定义、测量方法、暴露窗口。
- Comparator：非暴露、低暴露、参考水平。
- Outcome：疾病发生、死亡、复发、不良结局。

核心审稿问题：是否合并了调整后效应量？混杂因素是否足够相似？

### 2.3 诊断准确性问题：PIRD

- Population：疑似目标疾病的人群。
- Index test：待评价检测。
- Reference standard：金标准或临床参考标准。
- Diagnosis of interest：目标疾病/状态。

核心审稿问题：病例-对照诊断设计是否高估准确性？参考标准是否存在偏倚？阈值是否统一？

### 2.4 预测模型问题：PIRT

- Population：目标使用人群。
- Index model：模型、算法、预测变量。
- Reference/outcome：目标结局及时间窗。
- Timing：预测时间点和随访时间。

核心审稿问题：开发、内部验证、外部验证是否分开？是否报告校准度？是否存在数据泄漏？

## 3. Protocol设计

Protocol不是形式主义，而是防止事后选择性分析的关键文件。

### 3.1 Protocol必须预设

- 研究问题和目的。
- 纳入/排除标准。
- 数据库和灰色文献来源。
- 完整检索策略。
- 主要与次要结局。
- 效应量及其换算方式。
- 主模型：common-effect还是random-effects。
- 异质性处理策略。
- 亚组分析和Meta回归变量。
- 敏感性分析。
- 偏倚风险工具。
- 证据确定性评价方法。
- 数据和代码开放计划。

### 3.2 Protocol红线

- 不要写“根据异质性大小选择固定或随机模型”作为唯一规则。
- 不要等看到结果后再定义亚组。
- 不要把主要结局写得过多。
- 不要混合RCT和观察性研究作为同一个主分析，除非估计目标明确且有充分理由。

## 4. 检索策略

### 4.1 常用数据库

医学干预/诊断/预后问题通常至少考虑：

- PubMed/MEDLINE。
- Embase。
- Cochrane CENTRAL。
- Web of Science或Scopus。
- ClinicalTrials.gov、WHO ICTRP等注册平台。
- 领域数据库：CINAHL、PsycINFO、IEEE Xplore、CNKI、万方、VIP等。
- 灰色文献：会议摘要、学位论文、监管机构报告、预印本。

### 4.2 检索式构成

一个检索式通常由三类词构成：

1. 人群/疾病词。
2. 干预/暴露/检测/模型词。
3. 研究设计或结局词。

不建议在所有问题中都加入“meta-analysis/systematic review”等词，因为检索原始研究时会漏检。

### 4.3 检索报告要求

- 每个数据库的完整检索式。
- 检索日期。
- 时间范围。
- 语言限制及理由。
- 去重方法。
- 注册平台和灰色文献检索方法。
- 是否联系作者。

## 5. 文献筛选

### 5.1 推荐流程

1. 导入全部记录。
2. 自动去重。
3. 双人标题/摘要筛选。
4. 双人全文筛选。
5. 记录排除理由。
6. 第三人仲裁。
7. 生成PRISMA流程图。

### 5.2 常见排除理由

- 不符合人群。
- 不符合干预/暴露/检测。
- 无目标结局。
- 非目标研究设计。
- 重复数据。
- 数据不可提取。
- 仅摘要且无法获取足够信息。

## 6. 数据提取

### 6.1 基础信息

- 第一作者、年份、国家/地区。
- 研究设计。
- 招募时间。
- 样本量。
- 人群定义。
- 纳入排除标准。
- 随访时间。
- 资助与利益冲突。

### 6.2 干预/暴露/检测/模型信息

- 干预剂量、频率、疗程、依从性。
- 暴露测量方法、分组阈值、测量时间。
- 诊断检测阈值、操作者、设备、参考标准。
- 预测模型变量、算法、训练/验证设计、缺失值处理。

### 6.3 结局数据

- 二分类：事件数、总人数。
- 连续型：均值、SD、样本量。
- 生存结局：HR、95% CI、P值、Kaplan-Meier可提取信息。
- 诊断：TP、FP、FN、TN。
- 率：事件数、人时。
- 相关：r或rho及样本量。
- 预测模型：AUC/C-index、校准斜率、O:E、Brier score。

### 6.4 数据提取红线

- 不要把中位数和均值混合，除非预设转换方法并做敏感性分析。
- 不要把不同随访时间的结局直接合并。
- 不要把调整后和未调整效应量混合为同一主分析。
- 不要重复纳入同一队列的多篇文章。
- 多臂研究不得重复使用同一对照组而不处理相关性。

## 7. 偏倚风险评价

偏倚风险是针对“结果可信度”的评价，不等于报告质量评分。

| 研究类型 | 推荐工具 |
|---|---|
| 个体随机RCT | RoB 2 |
| 集群随机试验 | RoB 2 cluster version |
| 交叉RCT | RoB 2 crossover version |
| 非随机干预研究 | ROBINS-I |
| 非随机暴露研究 | ROBINS-E或领域适配工具 |
| 诊断准确性研究 | QUADAS-2 |
| 预测模型研究 | PROBAST / PROBAST+AI |
| 系统综述 | AMSTAR 2、ROBIS |
| 动物实验 | SYRCLE RoB |
| 患病率横断面研究 | JBI prevalence checklist / AXIS |

## 8. 是否合并：定量合成前判断

### 8.1 可以合并

- 研究问题一致。
- 数据结构一致或可合理换算。
- 研究设计相近。
- 偏倚风险可接受。
- 异质性有可解释来源。

### 8.2 不宜合并

- 研究对象、干预或结局定义根本不同。
- 关键数据不可换算。
- 测量方法差异导致效应含义不同。
- 极端偏倚风险研究主导结果。
- 单纯为了画森林图而合并。

## 9. 结果呈现

### 9.1 必备图表

- PRISMA流程图。
- 研究特征表。
- 偏倚风险图。
- 森林图。
- 异质性与敏感性分析表。
- 发表偏倚图/小样本效应检验。
- Summary of Findings表。

### 9.2 主文结果写法

应同时报告：

- 纳入研究数和样本量。
- 合并效应量及95% CI。
- 模型类型。
- τ²、I²、Q检验可作为辅助。
- 预测区间。
- 证据确定性。
- 临床解释，而不只是P值。

## 10. 结论写法

结论应按证据确定性表达：

- 高确定性：干预/暴露“很可能”产生某效应。
- 中等确定性：干预/暴露“可能”产生某效应，但进一步研究可能改变估计。
- 低确定性：效应不确定。
- 极低确定性：不能据此作出可靠结论。

不要写：

- “证明了”。
- “显著提高/降低，因此推荐临床应用”。
- “由于P<0.05，所以有效”。
- “没有统计学意义，因此无效”。
